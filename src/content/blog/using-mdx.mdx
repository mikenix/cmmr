Incubation projects die when momentum gets replaced by meetings.

At CMMR, we use a simple weekly rhythm to keep execution sharp:

## Monday: Alignment

- Confirm top outcomes for the week
- Resolve blocking decisions
- Assign clear owners

## Tuesday to Thursday: Delivery

- Build the smallest useful slice
- Demo interim progress
- Adjust scope based on new signal

## Friday: Review and Reset

- Review shipped outcomes
- Document lessons learned
- Set next week priorities

## Minimum Artifact Every Week

Every week should produce one concrete artifact:

- shipped code
- validated prototype
- decision memo
- technical risk retired

Execution rhythm is a compounding advantage. Keep it simple and keep it visible.
